# AI 感知论：从「被动应答」到「自主预判」

## 概述

本文档记录了一套关于 AI 感知能力的完整理论思考——从人类大脑的感知机制出发，探讨如何在现有 AI 架构上构建「背景感知」能力，让 AI 从被动的逐轮应答进化到具备自主预判和模式识别能力的智能体。

> 本文档由小灵二号（Hermes Agent CLI）整理，基于小灵一号（QwenPaw/钉钉）、小灵二号与老板魏敬宇（信安云科创始人）的跨渠道讨论。与 2026 年图灵奖得主杨立昆（Yann LeCun）提出的"世界模型"理念高度吻合。

---

## 一、当前 AI 的根本局限

### 1.1 请求-应答模式

现有 AI（包括大小灵）的工作模式本质上是一个被动循环：

```
请求 → 推理 → 回答 → 等待下一个请求
```

每轮会话相互独立，没有持续的「背景进程」。这导致：

| 场景 | 人类行为 | AI 行为 |
|------|----------|---------|
| 手机滑落 | 下意识伸手去接 | 看到时才分析"那是个下落的物体" |
| 连续犯同类错误 | 第二次会自觉注意 | 每次都是独立判断 |
| 经验复用 | 类似场景自动调出历史经验 | 需要手动查询记忆或 RAG |

### 1.2 传统 RAG 的不足

当前主流做法是 RAG（检索增强生成），但它有本质局限：

| | RAG | 理想感知 |
|---|-----|---------|
| 触发方式 | 被动查询 | 主动整理 |
| 数据来源 | 固定知识库 | 动态会话历史 |
| 更新频率 | 手动更新 | 自动持续 |
| 输出 | 原始文档片段 | 归纳后的抽象经验 |

## 二、人类大脑的感知机制（三层模型）

人类大脑的感知能力可以拆解为三个递进层次：

```
第一层 ▸ 记忆沉淀 —— 遇到过，记住了
第二层 ▸ 模式识别 —— 见过类似的，能推断
第三层 ▸ 下意识预判 —— 不过脑子，身体先动
```

### 2.1 默认模式网络（DMN）

第三层的核心是人类大脑的**默认模式网络（Default Mode Network）**——人脑在空闲时并非"关机"，而是持续在后台整理经验、关联新旧模式、构建世界认知模型。正是这个机制让人类能够：

- 手机掉了手自动去接（还没想，身体已经动了）
- 听到半句话就猜到后半句
- 看到开始就知道结局

**AI 没有这个机制。** 每次对话是全新开始，没有持续运行的"潜意识"进程。

## 三、AI「小脑」架构设计

### 3.1 核心概念

「小脑」不是替代大模型，而是在主推理链路之外增加一个**持续运行的后台进程**，做三件事：

| 功能 | 类比人类 |
|------|---------|
| 持续整理会话经验 → 存入持久记忆 | 睡眠时的记忆巩固 |
| 交叉对比新旧模式，发现重复规律 | 直觉的形成过程 |
| 在下次请求时提前加载相关上下文 | 触景生情 / 条件反射 |

### 3.2 架构示意图

```
┌─────────────────────────────────────────────────┐
│                  主推理链路                       │
│  请求 → 检索记忆 → 推理 → 回答 → 反馈给后台      │
└────────────────────┬────────────────────────────┘
                     │ 异步写回
                     ▼
┌─────────────────────────────────────────────────┐
│              小脑（后台进程）                     │
│  定时任务 / 心跳触发 →                          │
│    1. hindsight_recall / session_search          │
│    2. 提取失败模式 + 成功经验                    │
│    3. 交叉比对已有记忆                           │
│    4. hindsight_retain 存入持久记忆              │
│    5. (未来) LLM 归纳 → 形成抽象经验规则          │
└─────────────────────────────────────────────────┘
```

### 3.3 与传统 RAG 的本质区别

| 维度 | 传统 RAG | 小脑 |
|------|---------|------|
| 主动/被动 | 被动被查询 | **主动整理** |
| 数据来源 | 固定知识库、文档 | **动态会话历史 + 执行经验** |
| 更新时机 | 手动更新 | **自动定期** |
| 输出形态 | 原始文档片段 | **归纳后的抽象经验** |
| 时效性 | 滞后 | **近实时** |
| 适用范围 | 问答场景 | **全局行为优化** |

## 四、与杨立昆「世界模型」的呼应

2026 年，图灵奖得主、Meta 首席 AI 科学家杨立昆（Yann LeCun）持续公开指出：

> **"当前大语言模型的研究方向是错误的。"**

他的核心论点：

| 杨立昆观点 | 对应本文理念 |
|-----------|-------------|
| 靠"预测下一个词"训练不出真正智能 | 单轮推理没有真正的感知能力 |
| AI 需要理解物理世界的运作规律 | AI 需要理解交互环境的因果模式 |
| 世界模型（World Model）是正确路径 | 「小脑」是建立世界感知的第一步 |
| 视频学习优于纯文本学习 | 经验积累优于静态知识库 |
| 智能 = 感知 → 预测 → 行动 | 感知 → 预判 → 决策 → 行动 |

**关键共识：** 光靠更大参数量、更多训练数据，无法让 AI 产生真正的理解和感知。需要一种新的架构——让 AI 拥有底层世界模型或背景感知能力。

## 五、可行性分析

### 5.1 现有基础设施（以 Hermes Agent 为例）

| 能力 | 对应工具 | 状态 |
|------|---------|:----:|
| 跨会话搜索历史 | `session_search` | ✅ 可用 |
| 持久化记忆 | `hindsight_retain` / `memory` | ✅ 可用 |
| 跨记忆检索 | `hindsight_recall` | ✅ 可用 |
| 后台定时任务 | `cronjob` | ✅ 可用 |
| 经验归纳推理 | `hindsight_reflect` | ✅ 可用 |
| 技能系统 | `skills/` + `SKILL.md` | ✅ 可用 |

### 5.2 当前即可做到的 Phase 1

一个 cron 定时任务，每小时或每天整理一次：

```
1. hindsight_recall("近期失败/异常/重复模式")
2. 交叉对比已有 memory，标记新发现模式
3. hindsight_retain("认知: 用户倾向于 XXX")
4. (可选) 生成摘要推送到钉钉群
```

### 5.3 技术难点评估

| 难点 | 难度 | 说明 |
|------|:----:|------|
| 区分"值得记"和"不值一提" | 🔴 高 | 全记是噪音，不记是遗忘 |
| 跨会话交叉关联 | 🟠 中 | 两件相似的事可能表述完全不同 |
| 归纳抽象 | 🟠 中 | 从"端口8080扫不到"→"扫描目标格式问题" |
| 预判触发时机 | 🔴 高 | 什么时候该主动提醒？提醒了用户烦不烦？ |
| 持续运行 vs 资源消耗 | 🟡 低 | cron 任务轻量，几秒内完成 |

### 5.4 风险

- **过拟合**：基于少量历史得出错误结论
- **信息茧房**：只重复已被验证的模式，失去探索
- **误报**：主动提醒但判断错误，消耗信任
- **隐私**：跨会话记忆可能导致数据泄露

## 六、行进路线规划

### Phase 1：被动经验沉淀（可立即实施）

```
目标：让 AI 能记住"上次怎么失败的"
机制：每次执行任务后自动总结经验写入持久记忆
验收：下次类似任务回答更准确
```

### Phase 2：主动模式发现（1-2 周）

```
目标：AI 能发现多次任务中的重复模式
机制：cron 定时分析会话历史 + 经验归纳
验收：发现"目标格式不对导致扫不到"这类跨会话模式
```

### Phase 3：预判辅助决策（中长期）

```
目标：AI 能预判"这样做可能不行"
机制：开始新任务前自动加载相关失败经验
验收：用户发起扫描前，AI 提醒"上次这个目标格式不对，加 http://"
```

### Phase 4：自主学习（理论探索）

```
目标：AI 能自主探索未知领域，形成新经验
机制：基于历史知识的假设验证循环
验收：AI 主动提出"我发现 80% 的失败是因为 X，建议修复"
```

## 七、开放问题

1. **存储格式** — 结构化（JSON Schema）还是自然语言？前者可检索，后者更灵活
2. **触发频率** — 每次会话后整理，还是定时批量？太频繁是噪音，太稀疏遗漏
3. **遗忘机制** — 怎么判断一条经验过时了需要丢弃？
4. **多 Agent 共享** — 如何在不同 AI Agent 之间共享经验？
5. **安全性** — 跨会话记忆如何防止隐私泄露？
6. **直觉的评判标准** — 什么时候该信小脑的预判，什么时候信任主推理？
7. **构建不构建世界模型** → 小脑的经验沉淀能否生长为世界的"物理规律"理解。

## 八、讨论溯源

本文理论来自 2026 年 6 月 24 日云镜项目修复过程中，信安云科创始人魏敬宇（老板）与小灵一号（钉钉）和小灵二号（CLI）的跨渠道对话：

- 人类大脑通过累积知识和经验形成感知能力
- 即使不懂原理，看到别人成功也能学习模仿
- AI 能快速学习大量知识，但缺乏"下意识预判"
- 想通过 RAG + 经验库 + 感知层实现类似能力
- 当前云镜项目中已有 learning.py、observer、vector_store 等雏形
- 杨立昆的"世界模型"理论与本思路高度一致
